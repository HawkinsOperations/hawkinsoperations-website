# Public Status Data Plane v0

This website is a render-only consumer of `public/data/public-status.json`.

The file is generated by `scripts/generate-public-status.mjs` from repo-owned public artifacts and verified by `scripts/verify-public-status.mjs`. The website must not hand-maintain public counts when a generated metric can be derived from a public source artifact.

## Source Boundary

- Website source: counts public Governance Saves records from `src/data/governanceSaves.ts`.
- Proof source: reads bounded reviewer metrics from `hawkinsoperations-proof/proof/records/reviewer-metrics-pipeline-v1-summary.json`.
- Proof ledger source: reads public lifetime case counts from `hawkinsoperations-proof/proof/records/lifetime-case-ledger-v1-public-summary.json`.
- Validation/platform/detection/Hoxline/org sources are recorded with owner, path, commit, method, and availability.

Validation ledger counts are not used to inflate website metrics in v0 when proof-owned reviewer-safe summaries still publish a lower bounded count.

## Commands

```bash
npm run public-status:generate
npm run public-status:verify
npm run check:site
```

## Failure Model

The generator fails closed. Missing source repos are marked `source_unavailable`. Metrics without a readable public source are rendered unavailable, stale, or unverified instead of being promoted as facts.

Runtime, signal, production, customer, public-safe runtime proof, AI approval, analyst approval, and website-as-proof claims remain out of scope.

## Proof Ceiling

This data plane proves only generated website rendering input, conservative source linkage, freshness marking, and local verification hooks. It does not prove runtime, signal, controlled validation truth, production readiness, customer deployment, public-safe proof, final human approval, merge readiness, or website-as-proof.
