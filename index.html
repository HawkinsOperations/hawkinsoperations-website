Core thesis

AI is labor. Governance is authority.

AI can draft, compare, summarize, test candidate ideas, support implementation, and help reviewers find inconsistencies.

AI cannot promote truth, approve public claims, declare runtime state, or own evidence status. Human authorization controls final promotion.

Claim lifecycle

A public claim moves through gates.

  1. Source exists
  2. Static review
  3. Test defined
  4. Test validated
  5. Runtime active
  6. Signal observed
  7. Evidence linked
  8. Public safe

Truth boundaries

Separate claims before they become public proof.

01

Source truth

Counts: committed files or artifacts.

Not: deployment or test pass.

Allowed: source exists.

Blocked: detection is validated.

02

Runtime truth

Counts: deployed and enabled system state.

Not: repository text.

Allowed: runtime not proven.

Blocked: runtime-active.

03

Signal truth

Counts: observed telemetry or alert output.

Not: expected behavior.

Allowed: signal not observed.

Blocked: signal-observed.

04

Evidence truth

Counts: preserved, linked support.

Not: a claim summary.

Allowed: evidence pending.

Blocked: evidence-linked.

05

Public proof

Counts: reviewed and approved external wording.

Not: website presence.

Allowed: not public-safe.

Blocked: public-safe.

06

Legacy/reference truth

Counts: scoped V1 context.

Not: successor inheritance.

Allowed: may inform review.

Blocked: V1 proves V2.
